Resize by Pixels vs. Percentage

Pixel mode: Set exact width and/or height in pixels. Best when you need a specific output size (e.g., 1200×630 for Open Graph images).

Percentage mode: Scale the image relative to its original size (e.g., 50% reduces both dimensions by half). Best when you want to proportionally reduce file size.

Aspect Ratio Lock

Locking the aspect ratio ensures the image doesn't get distorted. When you change width, height adjusts automatically to maintain proportions. Always keep this on unless you intentionally want to stretch or squish the image.

Resize vs. Compress

Resizing changes the pixel dimensions. Compressing reduces file size at the same dimensions. For the smallest output, do both: resize to the display size first, then compress. Use the Image Compressor after resizing.

Frequently Asked Questions

Does resizing reduce file size?
Yes, significantly. Halving the dimensions reduces the pixel count by 75%, which dramatically reduces file size. For maximum reduction, compress after resizing.
Will resizing reduce image quality?
Upscaling (making larger) can introduce blurriness. Downscaling (making smaller) preserves quality well. Always start from the highest-resolution source.
Can I resize PNG with transparency?
Yes. The tool preserves transparency when outputting as PNG or WebP.
Is my image uploaded to a server?
No. All processing happens in your browser using the Canvas API. Your image never leaves your device.
